```ad-info
Der Prompt soll eine präzise und strukturierte Anleitung zur Entwicklung einer spezifischen Funktion liefern, indem er klare Anforderungen, technische Details, Implementierungsschritte und Qualitätskriterien definiert. Ziel ist es, eine effiziente, wartbare und qualitativ hochwertige Lösung zu erstellen, die alle funktionalen und technischen Vorgaben erfüllt.

**Führe jeden Schritt als eigenständigen Prompt aus.**
```
---
# Prompt
```
PROMPT ENGINEERING OPTIMIERUNG  

ZIEL:  
Entwickle eine präzise, effiziente und wartbare Implementierung für [spezifische Funktion].  

PRIMÄRE ANFORDERUNGEN:  
1. Anforderungsanalyse:  
   - Funktionale Anforderungen [MUST]  
   - Technische Rahmenbedingungen [MUST]  
   - Erfolgskriterien [MUST]  
   - Qualitätsmetriken [SHOULD]  

2. Technische Spezifikation:  
   - Input/Output-Formate [MUST]  
   - Performanceanforderungen [SHOULD]  
   - Sicherheitsanforderungen [MUST]  
   - Kompatibilitätsanforderungen [SHOULD]  

IMPLEMENTIERUNG:  
1. Architektur:  
   - Systemkomponenten [MUST]  
   - Datenflüsse [MUST]  
   - Schnittstellen [MUST]  
   - Technologie-Stack [MUST]  

2. Entwicklung:  
   - Modulare Struktur [MUST]  
   - Fehlerbehandlung [MUST]  
   - Logging-Strategie [SHOULD]  
   - Performance-Optimierung [SHOULD]  

3. Qualitätssicherung:  
   - Testabdeckung (min. 80%) [MUST]  
   - Code-Reviews [MUST]  
   - Performance-Tests [SHOULD]  
   - Sicherheitsaudits [MUST]  

DOKUMENTATION:  
1. Technische Dokumentation:  
   - API-Spezifikation  
   - Architekturübersicht  
   - Entwicklerdokumentation  

2. Benutzer-Dokumentation:  
   - Installationsanleitung  
   - Benutzerhandbuch  
   - Troubleshooting-Guide  

VALIDIERUNG:  
1. Überprüfung aller MUST-Kriterien  
2. Review der SHOULD-Kriterien  
3. Performance-Analyse  
4. Sicherheitsvalidierung  

FORMAT DER ANTWORT:  
- Strukturierte Gliederung  
- Priorisierte Auflistung  
- Klare Unterscheidung zwischen MUST/SHOULD  
- Konkrete Beispiele wo möglich  ```
